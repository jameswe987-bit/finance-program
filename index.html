功能说明 / 常用流程 / 注意事项
项目财务系统使用说明 先在“日常操作”录入流水和材料，再到“项目管理、财务管理、采购成本”查看统计和核对。

1. 日常操作

把常用录入口集中到一起，包括收支流水、进度款、账户调整、应收应付、采购入库、材料领用、分包流水、工程变更和人工记录。

适合每天录数据时使用，不需要到各个页面找按钮。新增后系统会自动跳到对应页面继续填写明细和上传图片。

2. 项目管理

管理项目合同额、预付款、保证金、进度款、预算、分包项目和工程变更签证。

项目完成后可以在“股份分红”里结算，结算会生成应付款，但真正扣银行账户需要录入付款流水。

3. 采购建议有什么用

采购建议是系统根据“材料计划、采购入库、材料领用、库存余额、预算单价”自动整理出来的提醒。

它主要告诉你：哪些材料库存不够、哪些材料还需要买、哪些采购价格可能超过预算、哪些项目材料计划还没补齐。

简单说，它不是采购订单，而是一个“采购提醒和成本预警清单”。你可以先看采购建议，再决定是否新增采购入库。

4. 财务管理

资金账户显示每个银行或现金账户余额；收支流水是总流水，支出、收入、应收应付付款都建议从这里录。

应收、应付是台账；如果在收支流水里关联应收/应付，系统会同步计算已收、已付和未收、未付。

5. 采购仓库和成本

采购入库记录买了什么、数量、单价、供应商和图片；材料领用记录出了多少；库存总览会自动算剩余数量。

成本总览会把材料、人工、分包、项目支出合起来看，方便判断项目是否超预算。

6. 图片识别和图纸分析

采购清单识别可以上传英文或图片清单，生成中文报告和明细表。图纸分析可以辅助识别材料用量线索。

这些属于外部接口功能，管理员可以在“账号权限”的接口开关里开启或关闭。

7. 云端同步

建议手机 APP 用来做现场日常录入，电脑端用来核对、导出和备份。手机录完后点击“上传本机数据”，电脑查看前点击“下载云端数据”。

目前上传和下载是手动操作，避免多台设备同时修改时互相覆盖。上传会先备份云端旧数据，下载会先备份当前设备数据。

8. 备份恢复

重要录入完成或更新程序前，建议进入“账号权限”的“本机备份与恢复”，导出完整备份文件。

如果后期数据异常，可以选择备份文件恢复。恢复只影响当前设备，需要同步到其它手机或电脑时再上传云端。

程序更新发布

GitHub + Netlify 自动发布流程
1. 修改程序 让 Codex 完成功能修改，并生成最新发布文件。
2. 上传 GitHub 把最新文件同步到 GitHub 仓库：jameswe987-bit / finance-program。
3. Netlify 自动部署 Netlify 会读取 GitHub 最新版本并自动发布；发布目录为根目录。
4. 手机刷新 APP 点击右上角“刷新”；如果看不到新版，再关闭后重新打开 APP。
打开 GitHub 打开 Netlify 部署 打开正式网站

注意：程序功能更新走 GitHub + Netlify；业务数据同步走 Supabase 云端，两者不是同一个功能。
